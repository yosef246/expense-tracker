# משימה: בניית אפליקציה אישית למעקב הוצאות

אני רוצה לבנות אפליקציה מובייל אישית לעקוב אחר ההוצאות היומיות שלי. זו אפליקציה רק לעצמי (לא לעוד משתמשים) — בלי הרשמה, בלי שרת, בלי סנכרון. כל הנתונים נשמרים מקומית בטלפון.

## טכנולוגיה
React Native + Expo. אחסון מקומי דרך AsyncStorage. שפה — עברית בלבד, כיוון RTL מלא. מטבע — שקל (₪) בלבד. עיצוב נקי ומודרני בסגנון iOS/Material, גרדיאנטים עדינים, פינות מעוגלות.

## מסכים (4 בלבד)

### מסך 1: בית
- כותרת עליונה: "💰 תקציב [חודש נוכחי] [שנה]" + סכום התקציב במספרים גדולים.
- בר התקדמות גדול שמראה אחוז ניצול תקציב חודשי. צבעי הבר משתנים אוטומטית: ירוק (#10b981) בטווח 0-49%, צהוב (#f59e0b) בטווח 50-89%, אדום (#ef4444) בטווח 90% ומעלה כולל חריגה.
- מתחת לבר: "הוצאת: X ₪" מצד אחד, "נשאר: Y ₪" מצד שני.
- כותרת "📅 הוצאות אחרונות" ומתחת רשימה של 5 ההוצאות האחרונות (תאריך + שעה קצרים + תיאור + סכום).
- כפתור "📊 היסטוריה" שמוביל למסך 4.
- כפתור עגול צף (FAB) עם "+" בפינה הימנית התחתונה — פותח את מסך הוספה.
- אייקון ⚙️ בפינה השמאלית העליונה — פותח את מסך ההגדרות.

### מסך 2: הוספת הוצאה
- כפתור "→ חזרה" למעלה.
- כותרת "הוצאה חדשה".
- שדה סכום (₪) — input מספרי גדול במרכז, מקלדת מספרית.
- שדה "על מה?" — input טקסט חופשי (דוגמה placeholder: "פחית קולה ביילו").
- שדה תאריך — date picker, ברירת מחדל = היום.
- כפתור "💾 שמירה" — שומר ב-AsyncStorage וחוזר למסך הבית עם הסכום הכולל מעודכן.

### מסך 3: הגדרות
- כפתור "→ חזרה" למעלה.
- כותרת "הגדרות".
- שדה "תקציב חודשי" (₪) — ברירת מחדל 2000, ניתן לעריכה.
- שדה "תאריך תחילת חודש" — dropdown (1 לחודש / 15 לחודש).
- כפתור "✅ שמירה".

### מסך 4: היסטוריה חודשית
- כפתור "→ חזרה" למעלה.
- כותרת "📊 היסטוריה חודשית".
- בורר חודש: חיצים ‹ ‹ משני הצדדים + שם חודש ושנה במרכז. ניתן לדפדף קדימה ואחורה.
- כרטיס סיכום: סה"כ הוצאות החודש הנבחר + תקציב + בר התקדמות באותה לוגיקת צבעים + אחוז ניצול.
- רשימה מלאה של כל ההוצאות של החודש הנבחר, ממוינות חדש למעלה. כל פריט: תאריך + שעה + תיאור + סכום.
- בתחתית — תיבת תובנות תכלת רכה עם: "ההוצאה הגדולה ביותר: [תיאור] ([סכום] ₪)" ו"ממוצע יומי: [סכום] ₪".

## מודל נתונים (AsyncStorage)
- מפתח "expenses": מערך של אובייקטים { id, amount, description, date (ISO), createdAt }.
- מפתח "settings": אובייקט { monthlyBudget, monthStartDay }.

## לוגיקה
- חישוב "הוצאות החודש" = סכימת כל ההוצאות בטווח התאריכים של החודש הנוכחי לפי monthStartDay.
- אחוז ניצול = (סה"כ הוצאות / תקציב) * 100.
- מחיקת הוצאה: סוואייפ ימינה על פריט ברשימה → כפתור מחיקה אדום → אישור.
- ממוצע יומי = סה"כ ההוצאות בחודש / מספר הימים שעברו בחודש (לחודש נוכחי) או מספר ימים בחודש (לחודש שעבר).

## דרישות איכות
- כל ההוצאות נשמרות גם אם האפליקציה נסגרת ונפתחת מחדש.
- ביצועים חלקים גם עם 1000+ הוצאות.
- אין שגיאה אם המשתמש מנסה לשמור הוצאה עם סכום 0 או ריק — תן הודעת אזהרה רכה.
- כל הסכומים מוצגים עם פסיק לאלפים (1,240 ₪) ושתי ספרות אחרי הנקודה לסכומים שיש בהם אגורות (8.90 ₪) או מספר שלם לסכומים עגולים (200 ₪).

## סדר עבודה מומלץ
1. הקמת פרויקט Expo חדש בעברית RTL.
2. מבנה ניווט בין 4 המסכים (React Navigation).
3. שכבת אחסון (AsyncStorage) + hooks לקריאה/כתיבה.
4. מסך הבית — תצוגה ובר התקדמות עם לוגיקת צבעים.
5. מסך הוספה — טופס ושמירה.
6. מסך הגדרות.
7. מסך היסטוריה עם בורר חודש.
8. עיצוב סופי + אנימציות עדינות.
