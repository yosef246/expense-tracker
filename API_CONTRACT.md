> and AsyncStorage keys. This document is the single source of truth for every frontend

## 1. TypeScript Interfaces

All types are exported from `frontend/src/types/index.ts`.

### `Expense`

Represents a single expense record.

```ts
export interface Expense {
  id: string;          // UUID v4, generated automatically by useExpenses.addExpense
  amount: number;      // Positive float, e.g. 8.9 or 200. Never negative. Never zero.
  description: string; // Free text, 0–200 characters. Empty string "" is allowed.
  date: string;        // YYYY-MM-DD — the user-chosen date of the expense (local time)
  createdAt: string;   // Full ISO 8601 timestamp, e.g. "2026-06-15T14:32:00.000Z"
}
```

**Field rules:**
- `id` — never set manually; always generated by `generateId()` inside `useExpenses`.
- `amount` — stored as a raw JavaScript `number`; do not format before saving.
- `description` — may be an empty string; enforce the 200-char maximum in the UI before calling `addExpense`.
- `date` — YYYY-MM-DD in **local** time. Use `dateToYMD(date)` to convert a `Date` object. Never use `Date.toISOString().slice(0,10)` (that is UTC and will be wrong for users east of UTC+0 at midnight).
- `createdAt` — set to `new Date().toISOString()` at save time; used as the sort key (descending).

### `Settings`

App-wide configuration stored in AsyncStorage.

```ts
export interface Settings {
  monthlyBudget: number;     // Positive integer (ILS). Default: 2000.
  monthStartDay: 1 | 15;    // Which day of the month the budget period resets. Default: 1.
}
```

**Field rules:**
- `monthlyBudget` — treat as a whole number (ILS). Values less than 1 are invalid and must be rejected in the UI.
- `monthStartDay` — the only valid values are the literal numbers `1` and `15`. Any other value violates the type contract.

### `BudgetPeriod`

Computed value returned by `getBudgetPeriod`. Never stored directly in AsyncStorage.

```ts
export interface BudgetPeriod {
  start: Date;   // Midnight local time on the period's first day (inclusive).
  end: Date;     // Midnight local time on the NEXT period's first day (exclusive).
  label: string; // Hebrew month + year, e.g. "יוני 2026".
}
```

## 2. Hooks

### 2.1 `useExpenses`

**Import:** `import { useExpenses } from '../hooks/useExpenses';`

Reactive interface over the AsyncStorage expense layer. All mutations update the in-memory `expenses` array immediately.

#### Signature

```ts
function useExpenses(): UseExpensesReturn
```

#### Return type

```ts
interface UseExpensesReturn {
  expenses:      Expense[];
  isLoading:     boolean;
  addExpense:    (payload: Omit<Expense, 'id' | 'createdAt'>) => Promise<void>;
  deleteExpense: (id: string) => Promise<void>;
  reload:        () => Promise<void>;
}
```

#### Fields

| Field | Type | Description |
|---|---|---|
| `expenses` | `Expense[]` | Full list, sorted descending by `createdAt` (newest first). Starts as `[]`. |
| `isLoading` | `boolean` | `true` while any async storage operation is in flight. Starts as `true`. |
| `addExpense` | `async fn` | Persists a new expense. Generates `id` and `createdAt` automatically. Re-throws on error so the caller can abort navigation. |
| `deleteExpense` | `async fn` | Removes an expense by `id`. Shows a native Alert for confirmation before removal is final. Catches errors internally and shows an Alert. |
| `reload` | `async fn` | Re-reads expenses from AsyncStorage. Must be called on mount and inside `useFocusEffect` on every screen that displays expense data. |

#### Usage example

```tsx
const { expenses, isLoading, addExpense, deleteExpense, reload } = useExpenses();

// Reload when screen gains focus
useFocusEffect(
  useCallback(() => { reload(); }, [reload])
);

// Add an expense
await addExpense({
  amount: 49.9,
  description: 'Lunch',
  date: '2026-06-16',
});

// Delete an expense
await deleteExpense('some-uuid-v4');
```

#### Error behaviour

| Operation | On failure |
|---|---|
| `reload` | Shows Alert "לא ניתן לטעון את ההוצאות". Does not re-throw. |
| `addExpense` | Shows Alert "לא ניתן לשמור את ההוצאה. אנא נסה שנית." and **re-throws** — the calling screen must catch and abort navigation. |
| `deleteExpense` | Shows Alert "לא ניתן למחוק את ההוצאה. אנא נסה שנית." Does not re-throw. |

### 2.2 `useSettings`

**Import:** `import { useSettings } from '../hooks/useSettings';`

Reactive interface over the AsyncStorage settings layer. Loads settings automatically on mount.

#### Signature

```ts
function useSettings(): UseSettingsReturn
```

#### Return type

```ts
interface UseSettingsReturn {
  settings:     Settings;
  isLoading:    boolean;
  saveSettings: (updated: Settings) => Promise<void>;
}
```

#### Fields

| Field | Type | Description |
|---|---|---|
| `settings` | `Settings` | Current settings. Initialized to `DEFAULT_SETTINGS` (`{ monthlyBudget: 2000, monthStartDay: 1 }`) until the async load resolves. |
| `isLoading` | `boolean` | `true` during initial load and during any `saveSettings` call. |
| `saveSettings` | `async fn` | Persists a full `Settings` object and updates state. Re-throws on error so the caller can abort navigation. |

#### Usage example

```tsx
const { settings, isLoading, saveSettings } = useSettings();

// Save updated settings
try {
  await saveSettings({ monthlyBudget: 3000, monthStartDay: 15 });
  navigation.goBack();
} catch {
  // saveSettings already showed an Alert; just stay on screen
}
```

#### Error behaviour

| Operation | On failure |
|---|---|
| Initial load | Silently keeps `DEFAULT_SETTINGS`. App remains usable. |
| `saveSettings` | Shows Alert "לא ניתן לשמור את ההגדרות. אנא נסה שנית." and **re-throws** — the calling screen must catch and abort navigation. |

## 3. Utility Functions

### 3.1 `formatCurrency` / `formatAmount`

**Import:** `import { formatCurrency, formatAmount } from '../utils/formatCurrency';`

`formatAmount` is an alias for `formatCurrency`; both are identical in behaviour.

#### Signature

```ts
function formatCurrency(amount: number): string
function formatAmount(amount: number): string  // alias
```

#### Rules

- Whole integers: no decimal places.
- Numbers with a fractional part: exactly 2 decimal places.
- Thousands separator: comma (`,`) via `he-IL` locale.
- Appends ` ₪` (space + shekel sign) manually for cross-platform consistency.
- Input is **not** clamped — negative values produce a leading minus sign.

#### Examples

| Input | Output |
|---|---|
| `0` | `"0 ₪"` |
| `200` | `"200 ₪"` |
| `1240` | `"1,240 ₪"` |
| `8.9` | `"8.90 ₪"` |
| `1240.5` | `"1,240.50 ₪"` |
| `-50` | `"-50 ₪"` |

### 3.2 `getBudgetPeriod`

**Import:** `import { getBudgetPeriod } from '../utils/getBudgetPeriod';`

Computes the budget period that contains a given reference date.

#### Signature

```ts
function getBudgetPeriod(
  monthStartDay: 1 | 15,
  referenceDate?: Date   // defaults to new Date() (today)
): BudgetPeriod
```

#### Logic

**`monthStartDay === 1`:**
- `start` = 1st of the calendar month containing `referenceDate`, midnight local.
- `end` = 1st of the following calendar month, midnight local.
- `label` = Hebrew name of `referenceDate`'s month + year.

**`monthStartDay === 15`:**
- If `referenceDate.getDate() >= 15`: period is 15th of this month → 15th of next month.
- If `referenceDate.getDate() < 15`: period is 15th of the previous month → 15th of this month.
- `label` = Hebrew name of the **start** date's month + year.

#### Examples

| `monthStartDay` | `referenceDate` | `start` | `end` | `label` |
|---|---|---|---|---|
| `1` | 2026-06-16 | 2026-06-01 | 2026-07-01 | "יוני 2026" |
| `15` | 2026-06-16 | 2026-06-15 | 2026-07-15 | "יוני 2026" |
| `15` | 2026-06-10 | 2026-05-15 | 2026-06-15 | "מאי 2026" |

### 3.3 `toYMD`

**Import:** `import { toYMD } from '../utils/getBudgetPeriod';`

Converts a `Date` to a `YYYY-MM-DD` string in **local** time. Used for safe lexicographic comparison with `expense.date` strings when filtering expenses within a period.

#### Signature

```ts
function toYMD(date: Date): string
```

#### Example

```ts
toYMD(new Date(2026, 5, 16)) // → "2026-06-16"
```

### 3.4 `getProgressColor`

**Import:** `import { getProgressColor } from '../utils/getProgressColor';`

Returns a hex color for a budget-utilisation progress bar.

#### Signature

```ts
function getProgressColor(percentage: number): string
```

#### Thresholds

| Range | Color | Hex | Meaning |
|---|---|---|---|
| `percentage < 50` | Green | `#10b981` | Healthy |
| `50 <= percentage < 90` | Yellow | `#f59e0b` | Watch out |
| `percentage >= 90` | Red | `#ef4444` | Near limit or over budget |

Input is not clamped before color selection. A value of `150` correctly returns red. The ProgressBar component must clamp the visual fill width to `100%` independently.

#### Examples

| Input | Output |
|---|---|
| `0` | `"#10b981"` |
| `49` | `"#10b981"` |
| `50` | `"#f59e0b"` |
| `89` | `"#f59e0b"` |
| `90` | `"#ef4444"` |
| `150` | `"#ef4444"` |

#### `getDaysElapsed`

```ts
function getDaysElapsed(period: BudgetPeriod, today: Date): number
```

Returns the number of calendar days from `period.start` up to and including `today`, capped at `getTotalDays(period)`.

- If `today` is before `period.start`, returns `1` (guard against division by zero in daily-average calculations).
- If `today` is after `period.end`, returns `getTotalDays(period)`.

```ts
// period = { start: 2026-06-01, end: 2026-07-01 }, today = 2026-06-16
getDaysElapsed(period, today) // → 16
```

---

#### `getTotalDays`

```ts
function getTotalDays(period: BudgetPeriod): number
```

Returns the total calendar days in the period (`end - start` in whole days).

```ts
// June 1 → July 1
getTotalDays(period) // → 30

// June 15 → July 15
getTotalDays(period) // → 30

// May 15 → June 15
getTotalDays(period) // → 31
```

## 4. AsyncStorage Keys

**Import:** `import { EXPENSES_KEY, SETTINGS_KEY } from '../constants/storage';`

| Constant | Value | Stored type |
|---|---|---|
| `EXPENSES_KEY` | `"expenses"` | `JSON.stringify(Expense[])` |
| `SETTINGS_KEY` | `"settings"` | `JSON.stringify(Settings)` |

**Rule:** Never hardcode the string literals `"expenses"` or `"settings"` anywhere in the codebase. Always import and use these constants. All reads and writes to AsyncStorage must go through the storage layer modules (`expenseStorage`, `settingsStorage`), not by calling `AsyncStorage` directly from screens or hooks.

## 6. Invariants and Constraints

These rules must never be violated. Violations cause incorrect UI state, data corruption, or crashes.

### Data invariants

1. **`expense.date` is local time, not UTC.** Always produce it with `dateToYMD(date)` or `dateToYMD(toYMD(...))`. Never use `new Date().toISOString().slice(0,10)`.

2. **Expense filtering uses lexicographic string comparison.** To decide whether an expense belongs to a `BudgetPeriod`, compare YYYY-MM-DD strings directly:
   ```ts
   const ymd = toYMD; // from getBudgetPeriod
   const inPeriod = expense.date >= ymd(period.start) && expense.date < ymd(period.end);
   ```
   Do not convert to `Date` objects for this comparison — it is unnecessary and risks timezone bugs.

3. **`expenses` array is sorted descending by `createdAt`.** The storage layer (`expenseStorage`) is responsible for maintaining this order. Do not re-sort in components.

4. **`id` is always a UUID v4 string.** Never construct or mutate an `id` manually in a component or screen.

5. **`Settings.monthStartDay` is a discriminated union `1 | 15`.** No other value is valid anywhere in the codebase.

6. **`Settings.monthlyBudget` must be a positive integer.** Validate this in the UI before calling `saveSettings`.

7. **`Expense.description` may be empty.** Do not treat an empty string as invalid — the field is optional from the user's perspective.

8. **`Expense.amount` is stored as a raw number.** Do not format it (e.g. with `formatCurrency`) before passing it to `addExpense`. Formatting is for display only.

### Hook usage invariants

9. **Call `reload()` inside `useFocusEffect` on every screen that displays expense data.** Without this, navigating back from `AddExpenseScreen` will show stale data.
   ```tsx
   useFocusEffect(useCallback(() => { reload(); }, [reload]));
   ```

10. **`useExpenses` and `useSettings` are independent.** They do not share state. Each screen that needs both must call both hooks.

11. **`settings` from `useSettings` starts as `DEFAULT_SETTINGS` (`{ monthlyBudget: 2000, monthStartDay: 1 }`) until the async load resolves.** Always guard on `isLoading` before rendering computed values that depend on settings (e.g. budget period, progress percentage).

12. **`addExpense` and `saveSettings` re-throw on error.** Wrap calls in `try/catch` and abort navigation on failure — the hooks have already shown a user-facing Alert.

### Utility invariants

13. **`getBudgetPeriod` uses local time throughout.** Its `start` and `end` are local midnight `Date` objects — not UTC midnight.

14. **`getProgressColor` does not clamp its input.** Values above `100` are valid (over-budget). The ProgressBar component must independently clamp its visual fill to `100%`.

15. **`HEBREW_MONTH_NAMES` is 0-indexed** (index 0 = January). Always access via `date.getMonth()` which is also 0-indexed — never add or subtract 1.

16. **AsyncStorage keys must only be accessed via the storage-layer modules.** Direct `AsyncStorage.getItem('expenses')` calls in screens or hooks are forbidden. Use `loadExpenses`, `addExpense`, `deleteExpense`, `loadSettings`, `saveSettings` from the storage modules instead.
